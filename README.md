# プロジェクト「雪だるま」— Claude Code 自動収益システム

AIエージェント(Claude Code)が制作・運用のほぼすべてを行い、人間は承認と意思決定のみを行う個人収益プロジェクト。

## 目標
- 初月: 基盤構築+数千〜1.5万円 / 3ヶ月: 月5万円 / 6ヶ月: 月10万円超
- 初期費用1万円。収益の50%を再投資する雪だるまモデル
- 人間の作業: 初期設定2〜3時間+月1〜3時間

## 事業構成
- **柱A**: Etsyデジタル商品(printables / SVGバンドル)— 日本×英語圏の情報格差を差別化軸に
- **柱B**: Pinterest(流入装置)
- **柱C**: Gumroad開発者向け商品(このシステム自体の商品化)

## 3層構造
| レイヤー | 何をするか | いつ |
|---|---|---|
| 戦略 | `/weekly-strategy` — KPIを全部見て「集中3・捨てる3・仮説1」 | 月曜20分(人間同席) |
| 制作 | `/daily-production` — 品質保証付き量産(テンプレ5本+10点法レビュー) | 毎朝cron自動 |
| オペレーション | lint・台帳・KPI集計・規約監視 | 毎日cron自動 |

## 人間がやること
1. **初回**: `ops/runbooks/initial-setup.md` のチェックリスト(2〜3時間)
2. **週次**: 月曜の戦略会議20分+出品セッション15分(`/publish-assist`、Publishクリックは人間)
3. **月次**: `ops/runbooks/monthly.md`(入金確認・証憑、10分)

## 重要な原則
- **人間の承認なしに何も公開されない**(review/pending → approved → 人間がPublish)
- **Etsy AI開示は三重ゲート**(テンプレ強制埋め込み→lint→人間目視)。詳細: `templates/brand/ai-disclosure.md`
- **緊急停止**: `ops/STOP` ファイルを作ると全自動ジョブが停止

## 主要ファイル
- `CLAUDE.md` — 運用憲法(絶対ルール・KPI判定基準・レートリミット)
- `niches/active.md` — 注力ニッチと候補
- `strategy/weekly/` — 週次戦略会議の議事録
- `data/` — 売上・経費・KPIの台帳(確定申告の根拠資料を兼ねる)
