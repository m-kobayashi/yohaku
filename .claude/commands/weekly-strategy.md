# 週次戦略会議(戦略レイヤー)— 毎週月曜・人間同席20分

CEO業務の外注。売上・KPI・出品状況をすべて見て「集中3・捨てる3・仮説1」を確定する。

## 入力(すべて読み込む)
- `strategy/weekly/{今週}.md` の判定下書き(kpi-update が生成済みのはず。なければ `python3 ops/scripts/kpi-aggregate.py` を実行)
- `data/listings.csv` / `data/sales-ledger.csv` / `niches/scoreboard.csv` / `niches/active.md`
- `strategy/hypotheses.md`(検証中仮説)/ 前週の `strategy/weekly/*.md`
- `review/rejected/` の今週分(テンプレ改善の燃料)

## 手順
1. **KPI報告**: 前週比サマリーを人間に提示(3行以内の結論を先に)
2. **撤退/倍賭け判定**: CLAUDE.mdの基準と照合した機械判定を提示し、人間の承認を取る
3. **「次の1週間で集中すべき3つ」**: 数字の根拠付きで提案
4. **「捨てるべき3つ」**: 撤退ニッチ・やめる作業・凍結する商品。根拠付き
5. **「検証すべき仮説1つ」**: 検証方法と判定基準(何を測り、いつまでに、どの数字なら支持か)をセットで定義し `strategy/hypotheses.md` に追記
6. **テンプレ改善**: rejected頻出原因をテンプレ[C]に予防策として反映(編集まで行う)
7. **再投資判断**: 留保50%ルールに基づき、今週の再投資額と使途(CLAUDE.mdの優先順位)を提案→人間承認
8. **議事録**: `strategy/weekly/{YYYY-Www}.md` に決定事項・承認結果・宿題を記録
9. **重要な方針変更があれば** `strategy/decisions.md` にADR形式(背景/決定/根拠/影響)で追記

## 原則
- 人間の意思決定時間は10分以内に収まるよう、すべての提案に「推奨+根拠」を付ける(選択肢の羅列で丸投げしない)
- 撤退提案を遠慮しない。1件$0.20の世界では判断の遅さの方が高くつく
