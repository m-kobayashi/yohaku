# ニッチスキャン(週次・土曜) — 需要と供給の歪み発見

後発が勝つ唯一の方法は「需要があるのに供給が薄い歪み」をデータで見つけ、競合より速く並べること。

## 手順
1. **停止チェック**: `ops/STOP` があれば即終了
2. **候補生成(発散)**: 以下の4ソースから新規候補を最大10件(週上限)
   - Pinterestトレンド(trends.pinterest.com をWebFetch)で上昇キーワード→「印刷物・テンプレートに変換可能か」でフィルタ
   - Etsy検索のロングテール複合語(3〜4語。WebSearchで "etsy {seed} printable" などのサジェスト・記事を探索)
   - 季節先回り: 今日から8〜12週先のイベントを逆算(Etsy検索は6〜8週前に立ち上がる)
   - 差別化軸: 日本×英語圏の情報格差(Japandi・和柄・日本の手帳文化等)を優先的に試す
3. **除外フィルタ**: `templates/brand/banned-words.md` §3(商標・版権)に触れる候補は即破棄
4. **スコアリング(収束)**: 各候補を5軸で採点し `niches/scoreboard.csv` に追記
   - 需要(0-5)×2 / 競合の薄さ(0-5)×2 / AI制作適合度(0-5)×2 / 歪みシグナル(0-5)×3 / 季節タイミング(0-5)×1 = 50点満点
   - 歪みシグナルの基準: 検索上位にレビュー100未満の若いショップが3つ以上=後発でも入れる証拠。上位が全部レビュー1,000超なら0点(問答無用で捨てる)
   - eRank/EtsyHuntのアカウントがあれば検索ボリューム・競合数を照会(無料枠節約のため候補5個に絞ってから)
5. **合格ライン**: 32点以上 → `niches/active.md` の「候補」セクションに追加(注力への昇格は週次戦略会議で人間が承認)
6. **調査ログ**: `niches/research/{YYYY-Www}-{テーマ}.md` に根拠(URL・観察事実)を記録

## 原則
- 常時2〜3ニッチ並走。1ニッチに全リスティングを賭けない
- 「小さい」と「具体的」を混同しない。ニッチは (1)買い手は誰か (2)何の問題を解決するか (3)支払い意思 の3軸で定義する
