# 日次量産ジョブ(制作レイヤー)— トークン節約版

毎朝の自動実行。商品案の生成からpendingキュー投入まで。

## 最重要原則(過去にこれを破ってsession limit到達・3.5時間浪費・ゴミ16個散乱)
- **claudeでSVGを試行錯誤して描かない。生成は決定論スクリプトに必ず委譲する。**
- **ルートや商品フォルダに使い捨て.py(`gen_*.py`等)を作らない。**
- 1回の実行で**商品1件まで**(上限: 商品1件+ピン5件/日)。

## 手順
1. **停止チェック**: `ops/STOP` があれば即終了
2. **本日の計画**: `niches/active.md` を読み、本日作る商品を**1件**決める
   - 検証中ニッチを優先。出品済みでピン不足の商品があればピン生産に回す
3. **SVG生成(claudeは描かない・スクリプト実行のみ)**:
   - 和柄: `python3 tools/wagara_svg.py <商品>/design [パターン名]`
   - 家紋: `python3 tools/kamon_svg.py <商品>/design`
   - **必要パターンが未実装の場合のみ** tools/ のスクリプトに関数を1つ恒久追加してから実行する(使い捨て.py禁止)
4. **PNG化**: `qlmanage -t -s 800 -o <商品>/mockups <商品>/design/*.svg`
5. **品質チェック**: 生成PNGを1枚だけ目視し幾何が崩れていないか確認(崩れていればスクリプトの該当関数を修正)
6. **リスティング**: `templates/03-etsy-listing.md` で listing.md / meta.yaml 作成(claudeの文章作業)
7. **lint**: `bash ops/scripts/lint-listing.sh <slug>` で検証
8. **キュー投入**: rubric 7点以上+lint合格を `review/pending/{slug}.md` に。`data/listings.csv` 追記
9. **ログ**: `logs/{YYYY-MM-DD}-daily.md` に概要(生成数・所要・特記)

## トークン節約チェック(終了前に自己確認)
- [ ] SVGはすべてスクリプト生成で作った(その場限りのPythonを書いていない)
- [ ] ルート/商品に使い捨て.pyを残していない
- [ ] 生成は商品1件に収めた

## 禁止
- 人間の承認なしの外部公開(出品・ピン投稿・SNS投稿すべて)
- 使い捨てスクリプトの量産
