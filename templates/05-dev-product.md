# テンプレ05: 開発者向け商品(Gumroad)制作 — 柱C

## 工数上限(厳守)
週次バッチ1回分まで。柱A(Etsy)の日次生産を圧迫する場合は柱Cをスキップ。

## [A] 入力契約(欠落時は生成せず停止)
- `product_concept`: 本プロジェクトで実際に作って運用実績のある仕組みのみ商品化する(机上の空論商品は作らない)
  - 例: 品質保証付き量産テンプレシステム、Etsy向けlintスクリプト、KPI自動集計、cron×Claude Code運用キット
- `slug`: `dev-{type}-{連番3桁}`

## [B] ブランド核
- voice.md(開発者向けは簡潔・実証ベースのトーンに調整可)/ banned-words.md 適用
- 収益の誇大表現は絶対禁止(banned-words §2)。「実運用中のシステム」という事実のみ語る

## [C] 生成手順
1. **汎用化**: 自プロジェクトの実ファイルから、個人情報・収益データ・アカウント情報を除去した汎用版を作成
2. **再現性担保**: README に「前提環境 / セットアップ手順 / 動作確認手順」を含め、クリーンな環境を想定して手順を机上検証
3. **動作検証手順の同梱**: 買い手が5分で動作確認できる quick-start を必ず含める
4. **LP文章**: Gumroad商品ページ用(英語)。構成: 何が手に入るか / 想定ユーザー / 含まれるファイル一覧 / 前提条件 / FAQ / **AI開示文(ai-disclosure.md と同文)**
5. **価格**: $9〜29。内容物のファイル数と独自性で決定
6. **配置**: `products/gumroad/{slug}/` に成果物+lp.md+meta.yaml

## [D] セルフレビュー
- quality-rubric.md(技術品質=再現性採点を重点。READMEだけ読んでセットアップが完結するか)
- 合格 → `review/pending/{slug}.md` / 不合格 → rejected へ
- 12週売上ゼロ+流入<100/月で凍結(CLAUDE.md のKPI基準)
